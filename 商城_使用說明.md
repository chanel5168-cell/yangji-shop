### 2.1 商品管理

#### 看現有商品

左側選單 → **🏪 商品管理**

可以看到所有商品的列表:
- **內部編碼**(系統自動產生,例如 `FG-01-0001`)
- **名稱 / 售價 / 成本 / 庫存 / 狀態(草稿/上架)**

#### 新增商品

1. 點右上「**+ 新增商品**」
2. 填:
   - 名稱(必填)
   - 分類(選一個)
   - 成本 + 幣別(CNY/USD/TWD 都可以)
   - 售價(系統可以從成本自動算建議價)
   - 庫存(初始進貨量)
3. 上傳商品圖(可選,可貼網址或上傳檔案)
4. 按「💾 儲存」

#### 修改商品價格

直接點該商品 → 改價格 → 儲存。**自動同步到所有裝置**(電腦改 → 手機立刻看到新價)。

#### 大量改價格 (批次調整)

商品管理上方有「**批次調整**」按鈕。可以:
- 全部商品一次提高 10%
- 某類別降價
- 從 Excel 匯入新價

### 2.2 進貨流程(廠商把貨給你 → 入庫)

#### 步驟

1. 左側選單 → **🚚 進貨單管理** → **+ 新增進貨單**
2. 選廠商(7 家可選)
3. 填:
   - **幣別**(廠商給你報價的幣別)
   - **匯率**(系統自動帶當前匯率)
   - **運送方式**(海運 / 空運 / 件運)
   - **關稅 %**
   - **利潤 %**(進貨成本 → 你的售價要加多少 %)
4. 加商品: 每件商品填數量 + 進入哪個倉 + 哪個貨架
5. 按「💾 儲存(含發票/備註)」**先儲存草稿**(不會動庫存)
6. 收到貨後 → 點該進貨單 → **「✅ 確認入庫」**
   - 庫存自動 +
   - 貨架位置記錄下來
   - 若你勾「**自動算售價**」,商品的售價會用設定的計價公式自動更新

#### 對廠商付款

進貨單裡 → 「付款」按鈕 → 選付款方式:
- 現金
- 匯款(從哪個我方銀行戶頭出)
- 開支票(從支票本選一張)

可以分次付,系統會自動算「應付」剩多少。

### 2.3 訂單流程(客戶下單 → 出貨 → 收款)

#### 客戶下單(自動發生)

客戶從前台 `https://chanel5168-cell.github.io/yangji-shop/index.html` 加購物車結帳 → 後台會自動跳出新訂單。

#### 你看到訂單怎麼處理

1. 左側選單 → **🛒 訂單管理**
2. 看到新訂單(狀態: 待處理 pending)
3. **收款**:
   - 客戶現金 → 加付款記錄(現金 100%)
   - 客戶開支票 → 「客票」+ 填票號/銀行/票期
   - 客戶匯款 → 「匯款」+ 收到後標記
4. **揀貨**:
   - 列印出貨單(會顯示每件商品的貨架位置 A-1-上 之類)
   - 揀貨員依貨架位置取貨
5. **出貨**:
   - 訂單上點「出貨」→ 狀態變「已出貨 shipped」
6. **完成**:
   - 收款收齊 + 已出貨 → **系統自動轉「已完成 completed」**(你不用做任何事)

#### 客票兌現 (票期到了)

左側選單 → **📜 客票管理**

看到所有未兌現的客票,系統會提醒「**今天 / 7 天內 / 已逾期**」。

票期到了:
- 帶去銀行存入我方戶頭 → 點「✓ 兌現」→ 選存入哪個銀行戶頭
- 銀行說退票 → 點「⚠ 退票」→ 填退票原因
  - **訂單應收會自動回沖**(客戶又欠錢了)
  - 訂單狀態退回「已出貨」

#### 客票背書轉讓 (進階,把客票拿去抵廠商)

進階用法: 客票背書給廠商,抵應付帳款。
- 點客票 → 「📤 背書轉讓」
- 選哪個廠商
- 選對應的進貨單(系統會列出未付清的)

#### 4 級會員等級

| 等級 | 累積消費門檻 | 折扣 |
|------|------------|------|
| 一般會員 (👤) | 0 | 不打折 |
| 銀卡 (🥈) | NT$ 5,000 | 95 折 |
| 金卡 (🥇) | NT$ 20,000 | 9 折 |

**自動晉級**: 客戶下單後系統自動算累積消費,跨門檻自動升級。會員下次下單**自動套用折扣**(VIP 價優先;沒設 VIP 價就用等級折扣)。

### 2.5 促銷活動 (5 種類型)

#### 活動類型

| 類型 | 用途 | 範例 |
|------|------|------|
| 🌐 全館折扣 | 整個網站打折 | 週年慶 95 折 |
| 💸 滿額折扣 | 訂單滿 X 元才折 | 滿 1000 享 9 折 |

#### 怎麼設

左側選單 → **⚙ 系統設定** → **🎉 促銷活動** → 拉出 5 個按鈕:
- ＋ 全館折扣
- ＋ 滿額折扣
- ＋ 多件特價
- ＋ 滿額贈
- ＋ 滿額免運

點任一個 → 填名稱 + 條件 + 啟用 → 儲存

**多個活動可以共存**(全館 95 折 + 滿 1500 免運 + 滿額贈,通通自動疊加)。

### 2.8 倉庫管理

#### 多倉位 / 多貨架設計

3 個倉庫:
- 主倉(彰化) — A 區 4 個貨架(A-1 ~ A-4),每架 3 層(上/中/下)
- 二倉(伸港) — B 區 2 個貨架,每架 2 層
- 中部倉(台中) — C 區 2 個貨架,每架 2 層

商品**進貨時就分配到指定倉庫 + 貨架**,出貨單會自動列印貨架位置給揀貨員。

#### 倉間調撥(把主倉的貨移到二倉)

左側選單 → **🔄 倉庫管理 → 調撥** → 新增調撥單:
- 從哪個倉
- 到哪個倉
- 哪些商品 + 數量

調撥後總庫存不變,但位置移轉了。

## 🔧 第三部分:常見問題 / 故障排除

### Q. 我忘記後台密碼了

**A.** 4 種解法,從簡單到複雜:

1. **用其他角色帳號登入**(會計/揀貨員/業務任一個能登入即可)
2. **從另一台已登入的裝置進「後台帳號管理」改密碼**
3. **F12 開發人員工具 → Console 貼這行重設**(若你還能進其他電腦):
   ```
   localStorage.removeItem('shop_admin_login'); location.reload();
   ```
4. **最終手段**: 開 `https://chanel5168-cell.github.io/yangji-shop/seed-test.html` → 點「全清並注入」(會重設成預設帳號 admin@shop.local/admin123,**但會清掉所有資料**)

### Q. 客戶說網站打不開

**A.** 檢查:
1. 公網網址正確嗎? `https://chanel5168-cell.github.io/yangji-shop/`
2. 你公網的 GitHub Pages 還在嗎? 開:
   ```
   https://github.com/chanel5168-cell/yangji-shop/settings/pages
   ```
   如果顯示「Your site is live at...」綠字 = 沒事
3. 客戶的網路是不是有問題?
4. **你需要技術人員幫忙**就把網址 + 截圖傳給工程師看

### Q. 圖片不見了

**A.** 通常 2 種原因:

1. 商品的「images」欄位空白 → 後台編輯該商品 → 上傳圖
2. 圖片路徑錯誤(已自動修復過,理論上不會再發生)

### Q. 跑馬燈不跑

**A.** 後台 → 系統設定 → 找「**啟用頂部跑馬燈**」勾選。也要確認「公告文字」有填東西,或啟用中至少有 1 個促銷活動。

### Q. 雲端跟本地資料不一樣

**A.** 通常是其中一邊還沒同步。F5 刷新一下,等 1~3 秒。如果還不對:
1. 開 F12 → Console
2. 看有沒有紅色錯誤訊息
3. 截圖傳給工程師

### Q. 訂單收完款還是「待處理」狀態

**A.** 系統自動轉換需要兩個條件 **同時滿足**:
- 訂單已點「出貨」(status=shipped)
- 應收 = 0(全部款項都收齊了)

如果你只收款沒出貨,訂單會卡在「pending」。要點「出貨」按鈕推進。

### Q. 我改了商品價格但客戶看到舊價格

**A.** 客戶端可能有快取。讓客戶按 **Ctrl + Shift + R** 強制重整。或等 1~2 分鐘自動更新。
