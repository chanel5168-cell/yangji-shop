## 3. 系統架構

### 3.1 三層架構

```
┌─────────────────────────────────────────────────────┐
│                 表現層 (Vue 3 SPA)                   │
│  ┌──────────────────┐    ┌──────────────────┐       │
│  │   index.html     │    │   admin.html     │       │
│  │   (前台/購物)     │    │   (後台/管理)     │       │
│  └────────┬─────────┘    └────────┬─────────┘       │
│           │                        │                 │
└───────────┼────────────────────────┼─────────────────┘
            │                        │
            ↓ load()/save()          ↓ load()/save() + cloudSave()/onSnapshot()
            │                        │
┌───────────┼────────────────────────┼─────────────────┐
│           ↓     儲存層 (雙寫)        ↓                │
│  ┌──────────────────────────────────────┐           │
│  │  localStorage (本地副本,離線可看)     │           │
│  │  shop_products / shop_orders / ...   │           │
│  └──────────────────────────────────────┘           │
│                  ↕ 雙向同步                          │
│  ┌──────────────────────────────────────┐           │
│  │  Firebase Firestore (雲端真相)        │           │
│  │  products / orders / ... 12 個       │           │
│  └──────────────────────────────────────┘           │
└─────────────────────────────────────────────────────┘
            ↓
┌─────────────────────────────────────────────────────┐
│              認證層 (3 層,並存)                       │
│  ┌──────────────────┐  ┌──────────────────┐          │
│  │ 自製帳密         │  │ Firebase Auth    │          │
│  │ shop_adminUsers  │  │ (Email/Password) │          │
│  └──────────────────┘  └──────────────────┘          │
│  ┌──────────────────────────────────────┐           │
│  │ LIFF (LINE Login → 自動建會員)         │           │
│  └──────────────────────────────────────┘           │
└─────────────────────────────────────────────────────┘
```

### 3.2 資料流

**寫入流**(管理員改商品例):
```
admin 改價格 → v-model 更新 this.products[i].price
            → save(STORE.products, this.products) [寫 localStorage]
            → cloudSave('products', product)      [寫 Firebase]
            → 其他裝置 onSnapshot 收到 → 更新本地
```

**讀取流**(訪客看前台):
```
頁面載入 → data() {} 從 localStorage 讀 (離線可用)
        → mounted() → bootstrapFirebaseFromLocal() 首次推上雲
                   → migrateImagePaths() 修舊資料
                   → subscribeAll() 啟動 onSnapshot
        → 後續任何 collection 變動自動 push 到 this.* + localStorage
```

#### 6.4.1 銷售流程 (賣商城給其他公司)

商城本身設計是 SaaS 模式 — 可賣給其他五金/批發商複製使用。每家獨立部署:

**部署一家新客戶的步驟**:

1. **客戶建自己的 Firebase 專案**
   - https://console.firebase.google.com → 建專案如 `xxx-shop`
   - 啟用 Firestore (asia-east1, 測試模式)
   - 註冊 Web app → 拿 firebaseConfig

2. **客戶 fork 你的 GitHub Repo**(或你 clone 給他們)
   - admin.html / index.html 替換 firebaseConfig 為他們的
   - 設 Firestore Rules (參考 FIRESTORE_RULES.md)

3. **GitHub Pages 啟用**
   - 客戶的 repo Settings → Pages → 啟用 main 分支

4. **LINE 設定**(可選)

## 7. 進銷流程詳解

### 7.1 進貨流程

```
1. 後台 → 進貨單管理 → 新增
2. 選廠商(suppliers)、幣別、運送方式、關稅、利潤率
3. 加 items: { productId, qty, cost, targetWarehouse, targetShelf }
4. 儲存(savePurchase): 寫入 purchases collection,confirmed=false
5. 確認入庫(confirmPurchase):
   - 對每個 item,product.stockByWarehouse[wh] += qty
   - 寫入 product.shelfByWarehouse[wh] = targetShelf
   - product.stock = sum(stockByWarehouse)
   - 若 autoCalcSellPrice: 重算 product.price
   - 若 importAsDraft: 新進貨進 isActive=false
   - purchase.confirmed=true, confirmedAt=now
```

### 7.2 訂單流程

```
1. 客戶從前台加購物車(cart.localStorage)
2. 結帳填會員資料 → 建立 order doc
3. 對每個 item: product.stockByWarehouse[wh] -= qty
4. order.status='pending', payments=[]
5. 收款(savePaymentModal):
   - 現金/匯款 → 立即計入 paid
   - 客票/信用卡 → checkStatus='pending', 待兌現
6. 出貨(揀貨員 picker 角色):
   - order.status='shipped'
   - 列印揀貨單(含 shelfLocation)
7. 兌現(會計):
   - 客票 → cleared, 銀行戶頭增
   - 退票 → bounced, 應收回沖, status 退回 shipped
8. 完成(自動,maybeAutoComplete):
   - shipped + remain==0 → completed
```

### 7.3 客票管理

```
客戶開客票來付訂單
  ↓
新增客票: customerChecks.push({
  payerName, payerBankCode, payerBranchCode, payerAccountNo,
  amount, dueDate, status:'pending',
  fromOrderId, fromPaymentId
})
  ↓ 票期到 →
3 種去向:
  ① cleared:  存入我方銀行戶頭(bankAccountId)
  ② bounced:  退票,訂單應收回沖
  ③ endorsed: 背書給廠商抵應付(endorsedToSupplierId, endorsedForPurchaseId)
```

### 7.4 倉間調撥

```
transfers.push({
  fromWarehouse: 'wh-main',
  toWarehouse: 'wh-second',
  date, items:[{productId, qty}], note
})
  ↓
對每個 item:
  product.stockByWarehouse[from] -= qty
  product.stockByWarehouse[to]   += qty
  總 stock 不變(只是位置移轉)
```

## 8. 12 個歷史踩過的雷

詳見 `yangji-shop-skill/references/common-pitfalls.md`。最關鍵的 5 個:

### 雷 7. Vue 3 method vs computed
- **症狀**: template 用 `licenseStatus.kind` 直接讀,但 licenseStatus 寫在 methods → 拿到 function.kind = undefined
- **修法**: 屬性訪問必須用 computed,事件處理才用 method

### 雷 8. Firestore onSnapshot 不能 length>0 過濾
- **症狀**: 雲端清空時本地不會更新
- **修法**: 直接覆蓋本地,雲端是真相來源

### 雷 9. GitHub Pages 子目錄絕對路徑會 404
- **症狀**: `/assets/...` 在子目錄部署下找不到
- **修法**: 用相對路徑 `assets/...`,加 `migrateImagePaths()` 一次性遷移

### 雷 10. Firebase 首次連線會用空雲覆蓋本地
- **修法**: `bootstrapFirebaseFromLocal()` 在訂閱前先把本地推上去

### 雷 11. Bash 環境 SSL 憑證問題
- **症狀**: Claude 的 Bash 工具 curl/git push GitHub 失敗
- **修法**: git push 必須在使用者 PowerShell 跑
