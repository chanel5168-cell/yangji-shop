## 快速啟動

### 1️⃣ 啟動本地伺服器（**每次使用前必做**）

打開 **PowerShell**，輸入：

```powershell
cd C:\Users\User\Desktop\我的商城
python -X utf8 start-server.py
```

看到「暘基五金商城 - 本地伺服器」訊息就成功了。**這個視窗保持開著**伺服器才會一直跑。

要停伺服器：在那個視窗按 `Ctrl + C`。

### 2️⃣ 用瀏覽器打開

| 用途 | 網址 |
|------|------|
| 前台（顧客逛的）| http://localhost:8000/index.html |
| 後台（管理員用）| http://localhost:8000/admin.html |

⚠ **絕對不要雙擊檔案開**（file:// 協定會出問題）

### 3️⃣ 後台登入

預設帳號：`admin@shop.local` / `admin123`

⚠ 上線前請改密碼！（admin.html 內 `DEMO_ADMIN`）

## 專案架構

### 三層架構

```
🎨 表現層    index.html（前台）+ admin.html（後台）
              · Vue 3 via CDN（無 build 步驟）
              · 工業金屬風 UI（鋼藍 #1a2a4a + 黃銅金 #c9a95c + 鐵線網格）
💾 儲存層    localStorage（離線備援）+ Firebase Firestore（雲端可選）
              · 圖片：assets/imgs/ 本地檔案（透過 /upload 端點）
🔐 認證層    Firebase Authentication（Email/密碼）+ LINE LIFF（可選）
🌐 伺服器    start-server.py（本地 Python，含 /_proxy 與 /upload 端點）
```

### 載入的外部 CDN

| 函式庫 | 用途 |
|--------|------|
| Vue 3 | UI 響應式框架 |
| Firebase 9 (compat) | 雲端資料庫與認證 |
| SheetJS (xlsx) | 解析 Excel 檔 |
| JSZip | 抽取 xlsx 內嵌圖片 |
| OpenCC.js | 簡體 → 繁體中文轉換 |
| LINE LIFF SDK | LINE 內嵌應用支援 |

#### 工具列

```
[搜尋商品] [分類篩選 ⭐外貿新品優先] [＋新增商品] [⚖批次調整] [🗑刪除已篩選]
```

#### 表格欄位

```
[☑] [圖片] [商品名稱] [分類] [價格] [庫存] [狀態] [動作]
```

#### 批次操作

**多選**：
- 勾選任意商品（背景變黃銅金色）
- 跨頁、跨搜尋勾選會累加
- 表頭 ☑ 切換「全選/全不選」（依當前篩選結果）

**勾選後跳出黃銅金色批次列**：
```
已選 N 件 → [選擇要搬到的分類 ▼] [📦搬移] [🗑刪除已選] [清除選擇]
```

**⚖ 批次調整**（不勾選也可用）：
- 套用範圍：當前篩選 / 指定分類 / 全部商品
- 調整欄位：售價 / VIP 價 / 成本 / 庫存
- 調整方式：倍率（×N）/ 加減（+/- N）/ 設為固定值
- 四捨五入：取整 / 取 5 倍數 / 取 10 倍數 / 尾數 9
- 預覽前 5 筆 → 確認套用

#### 商品編輯欄位

- 商品名稱、分類、單位
- **售價**（前台預設）、**VIP 價**（個別優惠）、**成本**（admin only）
- 庫存、最低起訂量
- 圖片網址（每行一個 URL，可多張）
- 商品說明
- 規格（key-value 陣列）
- 上架/下架 切換

### 分類管理

```
[順序] [分類名稱] [父分類] [排序] [商品數] [動作]
[↑ ↓]  ⭐ 外貿新品  — 頂層 —    -1      161    [編輯][刪除]
[↑ ↓]  01 日用百貨  — 頂層 —     0      20     [編輯][刪除]
[↑ ↓]  └ 套裝掃把   01 日用百貨  0       0     [編輯][刪除]
```

- **↑↓ 上下移動**：與相鄰同層分類交換順序（自動更新 sortOrder 編號）
- **限制**：頂層只能跟頂層交換、子分類只跟同父的子交換
- **第一筆**的 ↑ 自動 disable、最後一筆 ↓ disable

### 訂單管理

- 搜尋訂單編號 / 收件人
- 狀態篩選：全部 / 待處理 / 已付款 / 已出貨 / 已完成 / 已取消
- 內聯下拉**直接改訂單狀態**
- 點「查看」開詳情：商品清單、收件資訊、付款方式、備註

### 進貨單

進貨流程：
1. **新增進貨單**：填日期、供應商、多項商品、單位成本
2. **儲存**為待確認狀態
3. 點 **✅ 確認入庫** → 自動把所有項目加進對應商品的庫存
4. 確認後不可再改，要改要刪除整單

頂部統計：本月進貨筆數 + 累計進貨金額。

### 會員管理

#### 工具列

```
[搜尋會員] [＋新增會員] · 共 N 名 · VIP X · 已停用 Y
```

#### 表格欄位

```
[姓名(備註預覽)] [Email] [電話] [等級] [角色] [下單數] [累積消費] [狀態] [動作]
```

**等級**自動依累積消費計算（用您設定的等級門檻）。彩色標籤顯示。

**角色**下拉切換：會員 / 管理員（管理員可登入後台）。

**動作**：
| 按鈕 | 功能 |
|------|------|
| 👁 詳情 | 跳出會員完整資訊 + 訂單歷史一次看 |
| ✏️ 編輯 | 改姓名、電話、地址、等級、角色、**📝 後台備註** |
| 🚫 停用 / ✓ 啟用 | 停用後不能登入 |

#### ➕ 新增會員（手動建立）

填 Email + 密碼 + 姓名 + 等級 + 角色 → 直接幫顧客建帳號（適合 VIP 大客）。

#### 後台備註

每個會員有個只有 admin 看得到的 `note` 欄位，可寫：
- 「常購大量、要 VIP 價」
- 「付款偏好轉帳」
- 「聯絡時段晚上」

備註前 15 字會在會員列表姓名下方以 📝 顯示提示。

#### 會員等級系統（自動晉級）

預設 4 級：
| 等級 | 折扣 | 累積消費門檻 |
|------|------|-------------|
| 👤 一般會員 | 100% | NT$ 0 |
| 🥈 銀卡會員 | 95%（9.5 折）| NT$ 5,000 |
| 🥇 金卡會員 | 90%（9 折）| NT$ 20,000 |
| 💎 白金會員 | 85%（8.5 折）| NT$ 50,000 |

**規則**：
- 顧客累積消費（不含取消訂單）達門檻 → 自動晉級
- 升級後**前台自動打折**（顧客看到的就是折後價）
- 個別商品的「VIP 價」優先於折扣百分比
- 自訂等級：到 系統設定 → 👑 會員等級與折扣 修改/新增/刪除

### 批次匯入

#### 4 步驟

1. **選擇匯入方式**
   - 拖曳檔案（支援 `.xlsx` `.xls` `.csv` `.json`）
   - 或直接貼上 JSON / CSV 文字

2. **匯率/價格倍數**
   - 預設 ×5（人民幣 → 台幣）
   - 一鍵：×1（不調）/ ×5（CNY→TWD）/ ×4.5

3. **上架到哪個分類**
   - 預設「⭐ 外貿新品」（覆蓋資料原始分類）

4. **預設值**
   - 預設庫存（例 100）
   - 預設單價（Excel 沒單價時用）
   - 預設單位（例「件」）

#### 自動處理

- ✅ **內嵌圖片自動抽取**（用 JSZip 解 xlsx 的 `xl/drawings/drawing1.xml`）
- ✅ **下載到本地**：透過 `/upload` 存成 `assets/imgs/{md5}.jpg`
- ✅ **自動繁體化**：簡體欄名 + 內容 → 繁體
- ✅ **欄位智慧對應**：

| Excel 欄名 | 對應到 |
|---------|------|
| 圖片 / image / images / imgUrl | 商品圖（內嵌或網址）|
| 貨號 / code / sku / 商品編號 | 商品編號 (tag) |
| 品名 / 商品名稱 / name / title | 商品名稱 |
| 價格 / 價格/元 / 成本 / 進價 / 批發價 | 成本（自動 ×倍率作為售價）|
| 單價 / 售價 / 零售價 | 售價 |
| 庫存 / 數量 / qty / stock | 庫存 |
| 單位 / unit | 單位 |
| 分類 / categoryId | 分類（會被覆蓋）|
| 說明 / 商品說明 / description | 商品說明 |
| 備註 / 備注 / 配件備註 / 包裝備注 | 自動合併進說明 |
| **其他未知欄位** | 自動轉成「規格」 |

#### 圖片自動壓縮（依數量）

| 圖片數 | 壓縮等級 | 每張約 |
|--------|---------|-------|
| ≤ 30 | 1000px / JPEG 85% | 60-90 KB |
| 31-80 | 800px / JPEG 82% | 35-50 KB |
| > 80 | 600px / JPEG 78% | 15-30 KB |

存到 `assets/imgs/{MD5_hash}.jpg`（自動去重 — 同張圖只佔一份）。

#### 預覽 + 確認

預覽表格：[圖] [名稱] [成本] [單價] [售價(×N)] [庫存] [單位]

點「✅ 確認匯入」後：
1. 自動繁體化文字
2. 下載圖片到 `assets/imgs/`
3. 商品的 `images` 變成 `['assets/imgs/abc.jpg']`
4. 寫入 localStorage（或 Firebase）
5. Console 顯示：`本地化中 30/161... ✓28 ✗2`

## 部署上線

### 啟用 Firebase（必要）

本地 localStorage 模式只能單機 demo。要正式營運必須接 Firebase：

1. 到 https://console.firebase.google.com/ 開新專案（免費）
2. 啟用 Authentication → Email/密碼
3. 啟用 Firestore Database
4. 取得 firebaseConfig 物件，貼進 `admin.html` 與 `index.html` 的 `firebaseConfig` 變數
5. 部署 Firestore Rules（README.md 有範例）

### GitHub Pages 部署

```powershell
cd C:\Users\User\Desktop\我的商城
git init
git add .
git commit -m "Initial yangji shop"
git remote add origin https://github.com/USER/yangji-shop.git
git push -u origin main
```

到 repo → Settings → Pages → Source 選 main → 啟用。

⚠ 注意：GitHub Pages **沒有** `/_proxy` 與 `/upload` 端點（只有靜態檔）。所以圖片本地化要在本地 Python server 跑完後再 push。

## 常見問題

### Q1：localhost 打不開、ERR_EMPTY_RESPONSE

**A**：伺服器沒在跑。開 PowerShell 執行：
```powershell
cd C:\Users\User\Desktop\我的商城
python -X utf8 start-server.py
```

### Q2：admin 看到資料、前台沒看到

**A**：兩種可能：
1. **localStorage origin 不一致**：別用 file:// 開（雙擊檔案），一律 `http://localhost:8000`
2. **前台分頁沒重整**：admin 寫入 localStorage 後，前台 Vue 不會自動更新，需要 F5

### Q3：改了程式但沒生效

**A**：瀏覽器快取。`Ctrl + F5` 強制重整（不是 F5）。我們的 `start-server.py` 有送 `Cache-Control: no-cache` 讓 F5 也行。

### Q4：商品價格全是 NT$ 0

**A**：Excel 沒有「單價」或「成本」欄。解決：
- Excel 加價格欄
- 或在「批次匯入」步驟 ④ 填預設單價
- 或匯入後用「批次調整」設售價

### Q5：簡體字沒轉繁體

**A**：F12 看 Console 是否有 OpenCC 載入訊息。沒網路會載不到 CDN。
測試：`OpenCC.Converter({from:'cn',to:'tw'})('长方形')` 應回傳 `'長方形'`。

### Q6：圖片上傳失敗

**A**：`/upload` 端點需要 Python 伺服器在跑。確認：
- PowerShell 視窗還活著（沒關）
- 看到「↑ upload XX KB → assets/imgs/...」表示成功

### Q7：批次匯入卡在「處理中 0/161」

**A**：可能是某張圖片下載超時。F12 Console 看錯誤。失敗的圖會保留原網址繼續處理。

### Q8：localStorage 容量不夠

**A**：本地模式約 5-10 MB。解法：
- 接 Firebase（無容量限制）
- 圖片用 `/upload` 存成檔案（不佔 localStorage）
- 分批匯入

### Q9：LIFF 設定後在 LINE 內開商城沒自動登入

**A**：
- F12 看 Console 訊息
- 確認 LIFF ID 正確（在 LINE Developers Console）
- 確認 LIFF 的 Endpoint URL 設成 `https://your-domain/index.html`

## 字典

- **LIFF**: LINE Front-end Framework（讓網頁在 LINE App 內運行）
- **OpenCC**: Open Chinese Convert（簡繁轉換函式庫）
- **localStorage**: 瀏覽器本地儲存，每個 origin 約 5-10 MB
- **Firestore**: Firebase 雲端 NoSQL 資料庫
- **CDN**: Content Delivery Network，加速載入靜態資源
- **CORS**: 跨來源資源共享，瀏覽器擋跨域請求的安全機制
- **base64**: 把二進位資料編碼成文字（圖片內嵌 HTML 用）
- **Webhook**: HTTP 回調網址，事件發生時推訊息
