## 🧠 已踩雷紀錄（不要再犯）

### 雷 1：file:// vs http://localhost 的 localStorage 不互通
業主一開始雙擊 admin.html 開（file://），導入了一些商品。改用 http://localhost 後資料「不見」— 其實在另一個 origin。

**規則**：一律用 http://localhost:8000。

### 雷 2：簡體字辨識
業主的 Excel 是簡體字（价格/货号/库存），doImport 一開始只認繁體字導致全部 0。

**規則**：欄位辨識同時支援繁簡（已加）。

### 雷 3：base64 圖片爆 localStorage
161 件商品 × 80KB base64 = 9.6 MB → 超過 5-10MB 限制。

**規則**：用 `/upload` 端點存成檔案，product.images 只存路徑（已實作）。

### 雷 4：re.sub 對 \n 的逃脫處理
Python 替換時 `\n` 被 re 模組當逃脫處理 → JS 字串內出現實體換行 → Vue 掛不上。

**規則**：用 `re.sub(pattern, lambda m: replacement, ...)` callable 形式。

### 雷 5：DATA_VERSION 遷移會覆蓋使用者資料
index.html 的 `migrateDefaults` 如果版本不符會用 SAMPLE_PRODUCTS 覆蓋使用者匯入的商品。

**規則**：bumping DATA_VERSION 時要小心 — 避免覆蓋使用者真實資料。已加保護（檢查現有 products 數量）。

### 雷 6：Vue.mount() 只能掛到一個元素
admin.html 有 login + admin 兩個 view，不能用 `mount('#a, #b')`。

**規則**：用單一 #app + v-if 切換子視圖。

### 雷 7：browser cache 嚴重影響開發
快取讓使用者一直跑舊版 JS。

**解決**：`start-server.py` 強制送 `Cache-Control: no-store, no-cache, must-revalidate, max-age=0`（已實作）。
